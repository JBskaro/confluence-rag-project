# Тестирование реальных запросов

## Проблема

FastMCP требует `session ID` для вызова инструментов, но механизм получения session ID через `initialize` не работает напрямую через HTTP.

## Решение

### Вариант 1: Использовать MCP клиент (РЕКОМЕНДУЕТСЯ)

Подключите MCP сервер к Open WebUI или Claude Desktop и выполните запросы через интерфейс.

### Вариант 2: Использовать bash скрипт с curl

Создан файл `test_queries.sh` - простой bash скрипт для тестирования.

**Запуск:**
```bash
chmod +x test_queries.sh
./test_queries.sh
```

### Вариант 3: Python скрипт (требует доработки)

Созданы файлы:
- `test_queries_auto.py` - упрощенный вариант
- `test_queries_final.py` - с обработкой session

**Проблема:** FastMCP требует session ID, но `initialize` не возвращает его в формате, который можно использовать.
